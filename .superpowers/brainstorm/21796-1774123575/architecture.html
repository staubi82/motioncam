Event-Flow: Bewegung erkannt

1. motion erkennt Bewegung

Kamera-Frame-Vergleich, Schwellwert überschritten

→
2. HTTP-Hook an Node

POST /api/hooks/motion-start

→
3. Node startet FFmpeg

child_process.spawn(), Recording läuft

→
4. motion-end Hook

POST /api/hooks/motion-end

→
5. Nachlaufzeit

Timer läuft, danach FFmpeg beendet

→
6. Post-Processing

FFprobe → Metadaten, FFmpeg → Thumbnail, SQLite-Eintrag, Mail
